## What is this dissertation about?

In plain English, each chapter answers one question:

**Chapter 1 — The tropical Pacific's "breathing pattern."** When the global climate warms, does the east-west circulation over the tropical Pacific (the *Pacific Walker Circulation*) get stronger or weaker? I use an isotope-enabled climate model (iCESM) to find an oxygen-isotope fingerprint that tells us which one is happening.

**Chapter 2 — When volcanoes explode, does El Niño follow?** Large tropical volcanic eruptions inject sunlight-blocking aerosols high into the atmosphere. Some scientists say this reliably triggers an El Niño event; others say the eruption only matters if ENSO was *already* leaning that way. I combine climate-model ensembles with coral-reef proxy records to adjudicate.

**Chapter 3 — California's coral connection to volcanoes.** A field follow-up to Chapter 2: do volcano-triggered ENSO events leave a fingerprint in Pacific coral proxies near the California coast? I look for correlations between known eruption dates and isotope signals in these records.

**Chapter 4 — Biogeochemistry meets fish populations.** _(working title — scope still being finalized)_ How do ocean biogeochemical cycles — the nutrients, oxygen levels, and acidity that fish care about — vary geographically, and what does that mean for fish-population ecology?

**Chapter 5 — How does the news talk about climate technology?** Using 10 years of global news coverage (from the GDELT database) and natural-language-processing tools, I measure how media sentiment about three climate-technology approaches — nature-based solutions, carbon-dioxide removal, and solar radiation management — has shifted, and what that might mean for public support.

**I'm a committee member — I just want to read the dissertation.**
Open the [live site](#) (once published). Or clone the repo and open `_book/index.html` locally after running `quarto render`.

**I want to read one specific chapter.**
Open the `.qmd` source directly on GitHub — it renders as a readable page even without cloning:

**I want to see the slideshows.**
Each chapter has two decks inside `chapter-N-*/presentations/`:
- `chapterN-slides.qmd` — a curated figure deck for talks
- `chapterN-progress.qmd` — a living timeline of progress, auto-updated nightly

**I want to reproduce the analysis.**
See *Build locally* below. Every figure regenerates from source; every number in the prose is pulled from a `stats.json` generated by `code/compute_stats.py`.

**I'm here for the code.**
The Chapter 2 analysis code lives in a sibling folder on the author's Mac at `~/Documents/Claude/Projects/Cladue/volcano enso/` and is referenced from this repo via the relative path `../../../volcano enso` in `.figures-sources.yml`. Other chapters' code is embedded directly inside each `chapter-N-*/code/` folder. The volcano-ENSO analysis will be mirrored into this repo (or published as a companion repo) as Chapter 2 approaches submission.

## Build locally

You need [Quarto](https://quarto.org/docs/get-started/) and Python 3.11+.

```bash
# One-time setup (macOS)
brew install quarto

# From this folder (the dissertation repo root):
./build.sh                              # full build — figures → book → slideshows → PDF
quarto preview                          # live-reload preview of the whole book
quarto preview chapter-2-volc-enso/chapter2.qmd   # preview one chapter
quarto render                           # build _book/ (HTML + PDF)
quarto publish quarto-pub               # publish to Quarto Pub
```

Under the hood, `build.sh` orchestrates three steps so you get both a book-style HTML site AND working reveal.js slideshows (Quarto's book mode would otherwise render slideshows as book chapters).

## How the "reproducible" part works

Two rules keep the website honest:

1. **Every number in the prose comes from `stats.json`, not a typed literal.** Chapter 2 paragraphs pull stats inline like `*p* < {python} val("ttest_p_threshold")` — if the underlying data changes, re-running `python3 code/compute_stats.py` updates `stats.json`, and the next Quarto render pulls in the new numbers automatically. Unfilled stats render as a loud `⚠TODO⚠` marker on the page, so placeholders never sneak through.

2. **Every figure is synced from a versioned analysis folder.** `sync_figures.sh` reads `.figures-sources.yml` and copies PNGs out of each project's analysis folder into the chapter's `figures/` directory. The script is idempotent (only copies newer files, never deletes).

Together these mean: the moment you re-run an analysis and re-render the book, every figure *and* every in-text statistic refreshes — no hand-editing required.

## Citation style

Default is [Chicago author-date](https://www.chicagomanualofstyle.org/tools_citationguide/citation-guide-2.html) (`csl: chicago-author-date.csl` in `_quarto.yml`). To target a specific journal, drop a CSL file in this folder (download from [zotero.org/styles](https://www.zotero.org/styles)) and override in the chapter's YAML header:

```yaml
---
title: "Chapter 2: ..."
bibliography: literature/bibliography.bib
csl: ../ametsoc.csl        # example: AMS style for submission to Journal of Climate
---
```
